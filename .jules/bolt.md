## 2024-05-18 - [Optimize ModelRunner `_convert_input_protos_to_python` overhead]\n**Learning:** Dynamic type introspection (like checking `__args__` or `__origin__` on `python_param_types`) inside the `_convert_input_protos_to_python` loop in `ModelRunner` is a significant performance bottleneck because it runs sequentially for every item in a stream or batch on the critical path.\n**Action:** Pre-compute and cache a mapped `cast_types` dict inside the `_MethodInfo.__init__` class during initialization, and use simple key lookup at runtime to eliminate the redundant reflection overhead.
